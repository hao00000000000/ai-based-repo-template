# Research Task Example

## Task Type

`research`

## Context

Current implementation supports a single external provider for a key integration; product roadmap may require multi-provider support.

## Goal

Produce a decision brief for introducing a second provider adapter.

## In Scope

- current code/path analysis
- required interface and config changes
- migration and rollout risk assessment

## Out of Scope

- implementation
- production rollout

## Acceptance Criteria

1. Brief includes at least two viable design options.
2. Trade-offs include complexity, reliability, and operational load.
3. Recommendation includes phased rollout and test plan.

## Required Checks

- confirm assumptions against current docs and code
- include unresolved unknowns explicitly

## Expected Artifacts

- architecture brief
- risk register
- recommended next implementation task list

## Final Report Format (Mandatory)

- `Summary`
- `Validation`
- `Risks`
- `Follow-ups`

## Risks and Escalation

- If required integration details are missing or undocumented, escalate with a precise question list.
