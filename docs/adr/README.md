# ADR Directory

This folder stores Architecture Decision Records.

Policy:

- ADR is mandatory for architecture-impacting changes.
- ADR must be created before implementation merge.

Naming convention:

- `NNN-short-kebab-title.md`
- Example: `001-polling-loop-vs-event-driven.md`

Lifecycle statuses:

- `Proposed`
- `Accepted`
- `Superseded`
- `Rejected`

Minimum required sections:

- Status
- Date
- Context
- Decision
- Alternatives Considered
- Consequences
- Rollout Plan
- References

Use `template.md` as the required base template.
